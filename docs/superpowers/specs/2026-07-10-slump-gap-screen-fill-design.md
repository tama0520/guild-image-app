# スランプグラフ空きコマへの液晶はめ込み — 設計書

作成日: 2026-07-10
対象: `streamlit_app.py`

## 目的

スランプ付き結果ポスト（まず【新宿歌舞伎町】かぶぱポストの結果）で、スランプグラフを
グリッド配置したときに最終行にできる空きコマへ、機種の液晶画像を1枚はめ込んで
見栄えを良くする。完成イメージはデスクトップの `111.jpg`（縦3列）・`222.jpg`（横4列）。

## 適用範囲と段階展開

1. **第1段階（本設計の実装対象）**: 新宿歌舞伎町「かぶぱポストの結果」のみに配線。
2. 第2段階: 上野新館・上野本館・新小岩・秋葉原のスランプ付き結果ポストへ横展開。
3. 第3段階: 記事用ページへ横展開。

描画関数の引数追加は全店舗共通（既定 None で従来挙動＝無害）。UI（液晶セレクタ）の
配線のみ `store == "新宿歌舞伎町"` に限定する。

## 描画ロジック（共通コア）

対象関数:
- `_attach_slump_to_table`（3列・縦レイアウト、かぶぱ等が使用）
- `_attach_slump_to_table_side`（4列・横レイアウト、16台以上の `_side`）

両関数に引数 `gap_screen_img: "Image.Image | None" = None` を追加する。

### 空きコマ判定

`n` = グラフ枚数、`rows = ceil(n / COLS)`、`empty = COLS * rows - n`。

| レイアウト | COLS | 起きうる empty | はめ込む条件 |
|---|---|---|---|
| 3列（縦） | 3 | 0, 1, 2 | empty == 2 |
| 4列（横 _side） | 4 | 0, 1, 2, 3 | empty in (2, 3) |

→ 共通ルール: **`empty >= 2` かつ `gap_screen_img is not None` のときだけ**はめ込む。
`empty` が 0 または 1 のときは何もしない（空きのまま）。

### はめ込み矩形（gap box）

グラフグリッドの余白 `PAD`（外周 12px）・`GAP`（グラフ間 8px）と揃える。
最終行の埋まっているグラフ枚数を `last_count = n - (rows - 1) * COLS` とすると:

- 左端 x0 = 最終行で最後のグラフの右端 + `GAP`
  = `PAD + last_count * cell_w + (last_count - 1) * GAP + GAP`（`_side` は `graph_x0` 起点を加味）
- 右端 x1 = グラフエリア右端 − `PAD`
- 上端 y0 = 最終行グラフの上端（`_attach_slump_to_table`: `th + PAD + (rows-1)*(row_h+GAP)`）
- 下端 y1 = y0 + `row_h`

box 幅 = x1 − x0、box 高さ = row_h。

### 液晶の配置（ド真ん中）

box 内にアスペクト比維持で収まる最大サイズへ液晶をリサイズし、
box の**横中央・縦中央**に配置する:

```
scale = min(box_w / img_w, box_h / img_h)
new_w, new_h = round(img_w*scale), round(img_h*scale)
offset_x = x0 + (box_w - new_w) // 2
offset_y = y0 + (box_h - new_h) // 2
```

液晶が横長で縦に余りが出ても上下中央に来る。背景は既存背景（`bg_path`）or 白のまま。

## 対象機種と液晶の決定

- 表が **1機種のみ** → その機種の液晶。
- 表が **2機種以上** → **差枚数が最大の台の機種**の液晶。
- 液晶画像は `get_machine_images(機種名)["screens"]`（1〜3枚）から取得。
- 該当機種の液晶が **未登録なら `None`**（はめ込まず空きのまま）。

差枚最大機種の判定には合成時に利用可能な `ban2diff` / `ban2mac`（各呼び出し箇所の
`_ban2diff_*` / `_ig_ban2mac_*`）を用いる。

## UI（A案・プレビュー後に画像ごとのセレクタ）

1. プレビュー生成時、各画像について上記ルールで **既定の液晶（`screens[0]`）を自動はめ込み**。
2. プレビューの各画像の下に expander「🖼️ 液晶画像を選ぶ」を設置。
   - 差枚最大機種の液晶 1〜3枚を **サムネ付き `st.radio`** で表示。
   - 「はめ込まない」選択肢も用意。
3. 選択は `st.session_state[f"_gap_screen_{store}_{fn}"]`（screen インデックス、-1=はめ込まない）
   に保存 → プレビュー再合成・実行時に反映。
4. 第1段階の配線は kabupa（`store == "新宿歌舞伎町"`）限定。

### ヘルパー

`_resolve_gap_screen(store, fn, table_machines, ban2diff, ban2mac) -> Image | None`:
- 対象機種を決定（1機種 or 差枚最大機種）。
- session_state のオーバーライド（選択 index）を読み、なければ `screens[0]`。
- 該当 screen 画像を開いて返す。未登録・「はめ込まない」なら `None`。

各 `_attach_slump_to_table(...)` 呼び出し（プレビュー・実行・記事用の計 7 箇所）で、
kabupa のときのみ本ヘルパーの戻り値を `gap_screen_img` に渡す。

## テスト観点

- 3列: n=4（empty=2）→はめ込み、n=5（empty=1）→空き、n=6（empty=0）→空き、n=7（empty=2）→はめ込み。
- 4列: n=5（empty=3）→はめ込み、n=6（empty=2）→はめ込み、n=7（empty=1）→空き。
- 1機種テーブル→その機種の液晶。多機種テーブル→差枚最大機種の液晶。
- 液晶未登録→はめ込まれず空きのまま（例外なし）。
- 液晶が box 内で縦横中央に配置される（アスペクト比維持）。
